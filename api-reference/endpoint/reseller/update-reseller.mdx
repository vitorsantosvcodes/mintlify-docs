## Como utilizar este endpoint?

Esse endpoint permite atualizar os dados de um revendedor existente.

### Headers obrigatórios

<ParamField
  header="Authorization"
  type="string"
  placeholder="Bearer {token}"
  required
>
  O token de autenticação no formato Bearer é obrigatório para acessar este endpoint.
</ParamField>

### Parâmetros da URL

<ParamField path="id" type="string" placeholder="136" required>
  ID do revendedor a ser atualizado. Esse parâmetro deve ser informado diretamente na URL.
</ParamField>

### Parâmetros do corpo da requisição (Payload)

<ParamField body="username" type="string" placeholder="" required>
  Novo nome de usuário do revendedor
</ParamField>

<ParamField body="password" type="string" placeholder="" required>
  Nova senha do revendedor.
</ParamField>

<ParamField body="email" type="string" placeholder="exemplo@email.com">
  Novo e-mail do revendedor. Campo opcional.
</ParamField>

<ParamField body="country" type="string" placeholder="Brasil" required>
  País de origem do revendedor
</ParamField>

<ParamField body="whatsapp" type="string" placeholder="+55 12312312312" required>
  Novo número de WhatsApp do revendedor.
</ParamField>

<ParamField body="telegram" type="string" placeholder="usuarioTelegram">
  Novo usuário do Telegram do revendedor. Campo opcional.
</ParamField>

<ParamField body="notes" type="string" placeholder="Observações">
  Observações adicionais sobre o revendedor. Campo opcional.
</ParamField>

### Exemplo de Requisição

```json
PATCH https://mcapi.knewcms.com:2087/users/136
Headers:
{
  "Authorization": "Bearer seu_token_aqui",
  "Content-Type": "application/json"
}
Payload:
{
  "username": "novo_usuario",
  "password": "nova_senha",
  "email": "novoemail@email.com",
  "country": "Brasil",
  "whatsapp": "+55 987654321",
  "telegram": "novoTelegram",
  "notes": "Atualização de informações do revendedor"
}
```
